# Блиц: Твой AI-стек

## IDE и AI-плагины

Основной инструмент — **Cursor**. Использую его как основную IDE благодаря глубокой интеграции с LLM (Composer mode для архитектурных правок и чат для контекстных запросов). Также использую **GitHub Copilot** как дополнение для предиктивного автодополнения кода «в потоке».

## Модель и под какую задачу

- **Claude 3.5 Sonnet / O1:** Основной выбор для написания бизнес-логики, сложного рефакторинга и генерации тестов. Отлично справляется с многошаговыми задачами.
- **GPT-4o:** Использую для быстрой проверки гипотез, написания простых скриптов или когда нужен более лаконичный ответ.
- **O1-mini:** Идеальна для отладки сложных алгоритмов и логических задач.

## Какие MCP используешь

Активно использую MCP (Model Context Protocol) для расширения возможностей AI:

- **Files-MCP:** Для глубокого анализа всей кодовой базы.
- **Google Search / Brave Search MCP:** Для поиска актуальной документации по библиотекам и API.
- **SQLite / Postgres MCP:** Когда нужно быстро проанализировать схему БД или сформировать сложный SQL-запрос.

## Cursor rules / системные инструкции

Да, использую `.cursorrules` для поддержания стандартов проекта. Самые полезные правила:

- **Architecture Enforcement:** Запрет на нарушение слоев (например, бизнес-логика не должна лезть напрямую в БД).
- **Code Style (DRY/SOLID):** Требование писать модульный, легко тестируемый код.
- **Production-Ready:** Автоматическое требование добавлять логирование, обработку ошибок и типизацию (Type Hints в Python).
- **Security First:** Правило на проверку уязвимостей (SQL injection, XSS) при написании нового кода.
